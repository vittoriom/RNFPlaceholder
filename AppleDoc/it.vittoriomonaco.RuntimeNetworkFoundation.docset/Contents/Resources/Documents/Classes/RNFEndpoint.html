initWithConfigurator:

Creates a new RNFEndpoint instance with a given RNFConfigurationLoader This method adopts lazy loading of the configuration. This means the RNFConfiguration is not actually loaded until the first method call comes to the instance

initWithName:

Creates a new RNFEndpoint instance with a given name This method eagerly loads the RNFConfiguration, using a RNFPlistConfigurationLoader This means the method will create a new instance of RNFPlistConfigurationLoader with the given name, throwing an exception if no configuration is found with the given name.

- (id)initWithName:(NSString *)name

Parameters

name

The name of the configuration to load, using a RNFPlistConfigurationLoader

Return Value

an RNFEndpoint instance

@throws RNFConfigurationNotFound if no plist configuration is found with the given name

Discussion
